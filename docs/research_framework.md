## 一、投资哲学

### 1.1 确定性的来源

期货市场是**零和博弈**（甚至负和），技术分析驱动的策略已被大量量化资金套利至失效。本系统追求**基于信息差的确定性**：

- **政策信息差**：部委政策从发布到市场完全 priced-in 存在时间窗口
- **基本面信息差**：产业链上下游的真实供需状况与盘面价格的背离
- **认知信息差**：对政策传导路径、产业链逻辑的深度理解

### 1.2 出手原则

- **只在高确定性时刻出手**：政策方向明确 + 基本面共振 + 产业链验证
- **不预测，只应对**：跟踪政策与数据的边际变化，而非预测价格
- **小亏大赚**：单笔亏损有限（止损明确），盈利让市场决定

## 二、分析框架

### 2.1 政策分析维度

#### 2.1.1 政策层级与影响力

| 层级 | 主体 | 影响力 | 典型文件 |
|------|------|--------|----------|
| 最高 | 国务院/中共中央 | ★★★★★ | 中央一号文件、政府工作报告 |
| 高 | 发改委、央行 | ★★★★☆ | 产业政策、货币政策 |
| 中 | 工信部、商务部、农业农村部 | ★★★☆☆ | 行业规划、进出口调控 |
| 专项 | 生态环境部、能源局、海关总署 | ★★★☆☆ | 环保限产、能源保供、关税调整 |

#### 2.1.2 政策类型与期货映射

1. **供给侧改革** → 黑色、有色（限产、产能置换、环保）
2. **粮食安全/收储政策** → 农产品（临储、进口配额、生物能源政策）
3. **能源安全/双碳** → 能源化工（保供、产能审批、碳排放）
4. **地产调控/基建刺激** → 黑色（螺纹钢、热卷）
5. **货币政策/汇率** → 贵金属、有色（美元定价品种）
6. **进出口政策/关税** → 农产品、有色（大豆、铜、镍）

#### 2.1.3 政策分析 checklist

- [ ] 政策发布的具体部委与层级？
- [ ] 政策是新增、强化、还是转向？
- [ ] 直接影响哪些品种？
- [ ] 产业链传导路径是什么？（上游→中游→下游）
- [ ] 市场对政策的预期程度？（超预期/符合预期/低于预期）
- [ ] 政策执行力度与落地时间？
- [ ] 历史类似政策的盘面反应？

### 2.2 基本面分析维度

#### 2.2.1 供需分析框架

**供应端**：
- 产能利用率、开工率
- 库存水平（厂库、社库、港口库）
- 进口/出口量与利润
- 天气/季节因素
- 政策约束（环保、安检、配额）

**需求端**：
- 下游开工率、订单情况
- 终端消费数据（地产、汽车、家电）
- 替代品价格与比价关系
- 出口竞争力

#### 2.2.2 产业链利润传导

```
原材料 → 中间品 → 成品 → 终端
```

- 监测各环节利润：利润向哪个环节集中？
- 利润异常往往是交易信号：如钢厂利润极低→螺纹钢相对铁矿石被低估
- 产业链套利机会：做多低利润环节、做空高利润环节

#### 2.2.3 库存周期定位

| 阶段 | 特征 | 策略倾向 |
|------|------|----------|
| 主动补库 | 需求↑ 库存↑ | 做多 |
| 被动补库 | 需求↓ 库存↑ | 做空 |
| 主动去库 | 需求↓ 库存↓ | 做空 |
| 被动去库 | 需求↑ 库存↓ | 做多 |

### 2.3 共振模型

高确定性信号 = **政策方向** × **基本面验证** × **产业链逻辑** × **市场预期差**

```
置信度评分 = Σ(政策权重 × 政策得分, 基本面权重 × 基本面得分, 
               产业链权重 × 产业链得分, 预期差权重 × 预期差得分)
```

## 三、信息处理流程

### 3.1 每日工作流程

1. **08:00-08:30** 政策监控：检查夜间及清晨各部委发布
2. **08:30-09:00** 基本面更新：库存、利润、基差数据
3. **09:00-09:30** Alpha 派深度研究：对重点品种/政策进行问答
4. **09:30-10:00** LLM 综合分析：生成交易信号与日报
5. **盘中** 持续监控突发政策与新闻
6. **收盘后** 复盘，更新框架

### 3.2 信息优先级

1. **P0 - 立即响应**：国务院重大政策、突发地缘政治、极端天气
2. **P1 - 当日研究**：部委级产业政策、重要宏观数据发布
3. **P2 - 本周研究**：行业规划、区域政策、产业链调研信息
4. **P3 - 持续跟踪**：常规数据更新、市场评论

## 四、交易信号框架

### 4.1 信号结构

```python
class TradingSignal:
    direction: str          # LONG / SHORT
    commodity: str          # 品种代码
    confidence: float       # 0-1 置信度
    catalyst: str           # 触发因素
    policy_driver: dict     # 政策驱动详情
    fundamental_driver: dict # 基本面驱动详情
    chain_logic: str        # 产业链逻辑
    entry_logic: str        # 入场逻辑
    stop_loss: str          # 止损逻辑
    target: str             # 目标逻辑
    holding_period: int     # 建议持仓天数
    risk_level: str         # LOW / MEDIUM / HIGH
```

### 4.2 信号过滤规则

- 置信度 < 0.7：不交易
- 政策与基本面方向矛盾：不交易
- 产业链逻辑不通顺：降低仓位
- 市场已充分预期（价格已大幅波动）：不追
- 临近交割月：不交易新信号

## 六、行业一页纸研究方法（基本面根基）

### 6.1 为什么需要行业一页纸

期货交易的核心是**理解标的所属行业的基本面格局**。缺乏行业认知的交易，本质上是在盲人摸象：
- 不知道产业链上下游的定价权分布
- 不知道行业的季节性、周期性规律
- 不知道政策的真实影响路径和时滞
- 不知道关键数据的来源、频率和可靠性

**行业一页纸 = 用一页纸建立对一个行业的系统性认知**，是交易决策的根基。

### 6.2 Alpha派 Mode 11（行业一页纸）

Alpha派 Agent Mode 11 可自动生成行业全景分析，覆盖：
- **行业概况**：市场规模、增速、竞争格局
- **产业链结构**：上游→中游→下游的供需关系
- **政策影响**：当前及预期政策对行业的冲击
- **核心数据**：关键指标、数据来源、跟踪频率
- **投资逻辑**：行业景气度判断框架

### 6.3 使用方法

```bash
# 生成单个行业
python scripts/generate_industry_research.py --industry 钢铁

# 批量生成所有行业（首次运行）
python scripts/generate_industry_research.py --all

# 只更新过期/缺失的行业（日常维护）
python scripts/generate_industry_research.py --update

# 预览将要生成的行业
python scripts/generate_industry_research.py --list
```

**调度**：每周一 08:00 自动运行 `--update`，确保行业认知不过时。

## 七、持续迭代机制

### 6.1 迭代触发条件

- 完成一笔交易（无论盈亏）
- 发现新的有效数据源
- 政策框架发生重大变化（如双碳→能源保供转向）
- 基本面分析模型失效

### 6.2 迭代内容

1. **更新 `docs/research_framework.md`**：修正分析框架中的认知偏差
2. **更新 `docs/commodity_chains/`**：新增/修正产业链逻辑
3. **更新 `docs/trade_cases/`**：记录交易案例与复盘
4. **更新 `config/settings.yaml`**：调整监控品种、阈值参数

### 6.3 复盘模板

见 `docs/trade_cases/_template.md`

## 八、风险控制

1. **单笔风险 ≤ 2% 账户资金**
2. **同板块仓位 ≤ 10%**
3. **政策转向信号出现时，立即平仓**
4. **基本面数据与预期背离时，重新评估**
5. **不交易不熟悉的品种**
